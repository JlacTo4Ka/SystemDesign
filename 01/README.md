# 1. Изучение задания

Целью работы является создание архитектурного описания системы хранения файлов в формате **Structurizr DSL**.

Система предоставляет API для управления пользователями, папками и файлами.

Основные операции системы:

- Создание нового пользователя
- Поиск пользователя по логину
- Поиск пользователя по маске имени и фамилии
- Создание новой папки
- Получение списка всех папок
- Создание файла в папке
- Получение файла по имени
- Удаление файла
- Удаление папки

Система должна обеспечивать хранение **метаданных файлов и папок**, а также **бинарного содержимого файлов**.

---

# 2. Роли пользователей и внешние системы

## Роли пользователей

### Пользователь (User)

Основной пользователь системы хранения файлов.

Основные задачи пользователя:

- регистрация нового пользователя
- поиск пользователей
- создание папок
- загрузка файлов
- получение файлов
- удаление файлов
- удаление папок

Пользователь взаимодействует с системой через **Web UI**, который отправляет HTTP-запросы к API системы.

---

## Внешние системы

### Notification Service

**Notification Service** — внешний сервис отправки уведомлений пользователям.

Он используется системой хранения файлов для информирования пользователей о результатах операций.

Примеры уведомлений:

- уведомление об успешной загрузке файла
- уведомление об удалении файла
- уведомление об ошибке операции

Уведомления могут отправляться:

- по **электронной почте (email)**
- через **push-уведомления**
- через **внутреннюю систему уведомлений**

Система взаимодействует с сервисом уведомлений через **HTTP API**.

# 4. Основные задачи пользователей и их реализация

## Создание нового пользователя

1. Пользователь отправляет запрос на регистрацию через **Web UI**
2. Web UI передает запрос в **API Gateway**
3. API Gateway направляет запрос в **User Service**
4. User Service сохраняет информацию о пользователе в **Metadata Database**

---

## Поиск пользователя

1. Web UI отправляет запрос через **API Gateway**
2. API Gateway передает запрос в **User Service**
3. User Service выполняет поиск пользователя в **Metadata Database**

---

## Создание папки

1. Пользователь отправляет запрос через **Web UI**
2. Web UI передает запрос в **API Gateway**
3. API Gateway направляет запрос в **Folder Service**
4. Folder Service сохраняет информацию о папке в **Metadata Database**

---

## Получение списка папок

1. Web UI отправляет запрос в **API Gateway**
2. API Gateway направляет запрос в **Folder Service**
3. Folder Service получает список папок из **Metadata Database**

---

## Создание файла

1. Пользователь загружает файл через **Web UI**
2. Web UI отправляет запрос в **API Gateway**
3. API Gateway направляет запрос в **File Service**
4. File Service сохраняет файл в **Object Storage**
5. File Service сохраняет метаданные файла в **Metadata Database**
6. File Service отправляет уведомление через **Notification Service**

---

## Получение файла

1. Web UI отправляет запрос в **API Gateway**
2. API Gateway направляет запрос в **File Service**
3. File Service получает метаданные из **Metadata Database**
4. File Service получает файл из **Object Storage**

---

## Удаление файла

1. Web UI отправляет запрос через **API Gateway**
2. API Gateway направляет запрос в **File Service**
3. File Service удаляет файл из **Object Storage**
4. File Service удаляет метаданные из **Metadata Database**
5. File Service отправляет уведомление через **Notification Service**

---

# 5. Перечень контейнеров

Контейнеры системы:

### Web UI

Клиентское веб-приложение системы.

Функции:

- пользовательский интерфейс
- отправка HTTP-запросов к API Gateway

Технологии:

Web Application (SPA)

---

### API Gateway

Точка входа для всех клиентских запросов.

Функции:

- прием HTTP-запросов
- маршрутизация запросов
- передача запросов внутренним сервисам

Технологии:

**C++ / userver**

---

### User Service

Сервис управления пользователями.

Функции:

- создание пользователей
- поиск пользователей

Технологии:

**C++ / userver**

---

### Folder Service

Сервис управления папками.

Функции:

- создание папок
- получение списка папок
- удаление папок

Технологии:

**C++ / userver**

---

### File Service

Сервис управления файлами.

Функции:

- загрузка файлов
- получение файлов
- удаление файлов
- отправка уведомлений

Технологии:

**C++ / userver**

---

### Metadata Database

База данных метаданных.

Хранит:

- пользователей
- папки
- метаданные файлов
- связи файлов и папок

Технологии:

**PostgreSQL**

---

### Object Storage

Хранилище бинарных данных файлов.

Хранит:

- содержимое файлов

Технологии:

**S3-compatible storage / MinIO**

# 9. Диаграмма Dynamic

Диаграмма показывает процесс **загрузки файла в папку**.

Последовательность:

1. Пользователь выбирает файл в Web UI
2. Web UI отправляет `POST /folders/{folderId}/files` в API Gateway
3. API Gateway проверяет папку через Folder Service
4. Folder Service обращается к Metadata Database
5. API Gateway передает файл в File Service
6. File Service сохраняет файл в Object Storage
7. File Service сохраняет метаданные в Metadata Database
8. File Service отправляет уведомление через Notification Service
9. API Gateway возвращает результат пользователю
